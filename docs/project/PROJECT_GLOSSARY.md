# PROJECT_GLOSSARY.md

Назначение: единый словарь терминов проекта.

## Термины

| Термин | Значение |
| --- | --- |
| `InternalProduct` | Внутренний товар компании. Ядро товарной модели, не равен карточке WB/Ozon. |
| `ProductVariant` | Вариант внутреннего товара: размер, цвет, исполнение или другая продаваемая модификация. |
| `MarketplaceListing` | Внешний листинг товара на WB/Ozon в конкретном магазине/кабинете. |
| `MarketplaceProduct` | Legacy-справочник marketplace products, сохранённый для совместимости старых операций. |
| `StoreAccount` | Магазин/кабинет маркетплейса в системе. |
| Cabinet | Кабинет продавца или управляемая сущность магазина, к которой привязаны операции и доступы. |
| Operation | Пользовательская операция: проверка, обработка, скачивание, загрузка, API/Excel workflow. |
| Run | Конкретный запуск операции или её шага. |
| Snapshot | Зафиксированный снимок внешних данных API/Excel на момент операции. |
| SyncRun | Запуск синхронизации внешних данных с внутренним справочником/листингами. |
| Excel flow | Операционный сценарий через загружаемый/скачиваемый Excel-файл. |
| API flow | Операционный сценарий через API WB/Ozon. |
| WB auto promotion | Автоматическая акция WB, по которой текущий API может отдавать summary/details, но не product rows через `promotions/nomenclatures`. |
| Ozon Elastic Boosting | Ozon акция `Эластичный бустинг`, реализованная в Stage 2.2. |
| Product Core | Ядро внутреннего справочника товаров: internal products, variants, listings, mapping, snapshots. |
| Object access | Ограничение доступа пользователя к конкретным объектам: магазинам, кабинетам, операциям. |
| Audit | Пользовательский и бизнес-аудит действий. |
| Techlog | Технический журнал событий, ошибок и диагностических кодов. |
| GAP | Зафиксированный пробел, противоречие или открытый вопрос. |
| ADR | Architecture Decision Record: принятое архитектурное решение. |
| Task-scoped package | Ограниченный пакет документов и контекста, который агент читает перед конкретной задачей. |
| Audit-gate | Обязательная проверка проектной документации или результата перед переходом к следующей фазе. |
