# docs/source

Назначение: ссылки на исходные входные материалы, которые остаются неизменяемыми источниками для проектирования и аудита.

## Файлы

| Источник | Расположение | Статус |
| --- | --- | --- |
| Итоговое ТЗ, редакция 25.04.2026 | `itogovoe_tz_platforma_marketplace_codex.txt` | Источник истины для аудита и спорных требований |
| Стартовые правила оркестрации | `promt_start_project.txt` | Исходный orchestration prompt |

Исходные TXT оставлены в корне, чтобы не менять источник истины и не ломать внешние ссылки/историю передачи. Исполнительные документы в `docs/` не заменяют итоговое ТЗ.

## Правило чтения ТЗ

По умолчанию агент не читает итоговое ТЗ целиком. Оркестратор выдаёт конкретные разделы ТЗ только когда они нужны для задачи, аудита спорного требования или закрытия gap.
