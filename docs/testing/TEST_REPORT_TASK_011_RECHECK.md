## Audit FAIL Findings Rechecked

1. Production/default resolver path can succeed.
   - Current code resolves only deterministic local `env://ENV_VAR_NAME` refs through `default_secret_resolver`.
   - Existing test patches the environment and production `WBApiClient` class, calls `check_wb_api_connection(...)` without injected resolver, reaches `active`, and verifies the sentinel token is not present in connection metadata, audit safe fields, or techlog safe fields.
   - Status: pass.

2. `active` cannot be manually submitted.
   - `ConnectionBlockForm` no longer exposes `status`.
   - `save_connection_block` rejects `status=active` with `ValidationError`.
   - POSTing an extra `status=active` field through the edit route does not activate the connection; status remains `configured` and `last_check_status` remains blank.
   - Status: pass.

3. Ozon/non-WB stores are denied.
   - `connection_edit`, `connection_check`, and `save_connection_block` enforce WB-only access for `module=wb_api`.
   - Store card hides WB connection controls for Ozon store.
   - Existing legacy Ozon `wb_api` row cannot be edited or checked through UI/service paths.
   - Status: pass.

4. Edit form is write-only for protected refs.
   - `protected_secret_ref` uses `PasswordInput(render_value=False)` and initial value is blank for existing instances.
   - GET edit form does not render the saved ref.
   - Blank submitted secret input preserves the old ref; non-blank input replaces it and leaves status `configured`.
   - Status: pass.

5. `__pycache__` outside `.venv`.
   - Before test execution: pass, no `__pycache__` found.
   - After test execution: remark, Django/Python test runs regenerated ignored `__pycache__` directories under `apps/` and `config/`.
   - Status: pass for executor handoff state, remark for tester-generated runtime artifacts.
