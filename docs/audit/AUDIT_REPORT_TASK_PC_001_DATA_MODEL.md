## Остаточные Риски

- Валидация cross-context инвариантов (`MarketplaceListing` vs `StoreAccount`, snapshot vs sync run/listing/operation) реализована через `clean()`. ORM `objects.create()` не вызывает `full_clean()` автоматически. Это не блокер TASK-PC-001, но до sync/import service layer нужно закрепить единый путь создания записей или добавить save/service validation tests.
- `ProductIdentifier.value` нормализуется в `clean()`, но прямое создание через ORM может обойти strip/required-normalized-string правило. Рекомендуется усилить тестами или сервисным контрактом до import/mapping задач.
- В рабочем дереве присутствует untracked `apps/product_core/` including `__pycache__/` files. Перед коммитом/передачей результата нужно не включать generated bytecode.
