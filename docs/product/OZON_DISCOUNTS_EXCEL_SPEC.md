# OZON_DISCOUNTS_EXCEL_SPEC.md

Трассировка: ТЗ §16-§17, §24.

## Назначение

Ozon -> Скидки -> Excel обрабатывает один Excel-файл акции формата `Эластичный бустинг`. На этапе 1 система не рассчитывает "скидку в процентах" как отдельный результат.

По строке определяется:

- участвует ли товар в акции;
- если участвует, какую итоговую цену акции записать.

## Входной состав

Один Ozon run принимает:

- ровно 1 файл `.xlsx`;
- книга должна содержать лист `Товары и цены`;
- строки данных читаются начиная со строки 4;
- строки 1-3 считаются служебной частью шаблона.

## Обязательные колонки

Обязательны колонки по буквам:

- J - минимально допустимая цена;
- O - минимальная буст-цена;
- P - максимальная буст-цена;
- R - остаток;
- K - признак участия в акции;
- L - итоговая цена акции.

## Нормализация

Для J, O, P, R:

- привести к нормализованному числовому виду;
- нечисловое значение считать отсутствующим;
- логика решения использует только нормализованные значения.

## Разрешённые изменения

Process может менять только:

- K;
- L.

Запрещено менять J, O, P, R, любые другие колонки, структуру шаблона, листы книги и служебные области.

## Каноническая логика строки

Правила применяются строго по порядку:

| # | Reason | Условие | K | L |
| --- | --- | --- | --- | --- |
| 1 | `missing_min_price` | J отсутствует | пусто | пусто |
| 2 | `no_stock` | R отсутствует или <= 0 | пусто | пусто |
| 3 | `no_boost_prices` | O и P одновременно отсутствуют | пусто | пусто |
| 4 | `use_max_boost_price` | P присутствует и P >= J | `Да` | P |
| 5 | `use_min_price` | P присутствует, O присутствует, P < J, O >= J | `Да` | J |
| 6 | `below_min_price_threshold` | O присутствует и O < J | пусто | пусто |
| 7 | `insufficient_ozon_input_data` | иначе | пусто | пусто |

Порядок правил менять нельзя.

## Запись в ячейки

K:

- если товар участвует, записывается строго `Да`;
- если не участвует, пустая ячейка.

L:

- если есть итоговая цена акции, числовое значение;
- если не участвует, пустая ячейка.

Запрещено писать в K что-либо кроме `Да` или пусто, писать в L текстовые суррогаты чисел, записывать `0` вместо пустого значения.

## Check Ozon

Check обязан:

- проверить наличие ровно одного файла;
- проверить лист `Товары и цены`;
- проверить обязательные колонки по буквам;
- проверить пригодность workbook к безопасной обработке;
- прочитать строки с 4-й строки;
- нормализовать J, O, P, R;
- определить решение по каждой строке;
- сформировать summary audit;
- сформировать detail audit;
- сформировать краткий итог;
- не создавать output file;
- ничего не записывать в workbook.

## Process Ozon

Process обязан:

- выполняться только по допустимой check-основе;
- учитывать актуальность проверки по `docs/product/OPERATIONS_SPEC.md`: marketplace, store/account, набор входных файлов, конкретные file versions, применённые параметры, business logic version;
- если актуальной успешной проверки нет, сценарий "Обработать" сначала создаёт отдельную check operation и только при допустимой основе продолжает process;
- открыть workbook;
- проверить лист и колонки;
- проверить возможность безопасного сохранения output workbook;
- выполнить логику решения по каждой строке;
- записать только K и L;
- сохранить output workbook;
- сформировать summary audit;
- сформировать detail audit;
- сформировать краткий итог.

## Критические ошибки

Минимально:

- файл не загружен;
- загружено более одного файла;
- книга повреждена;
- отсутствует лист `Товары и цены`;
- отсутствуют обязательные колонки по буквам;
- невозможность сохранить выходной файл.

## Отличие от WB

На этапе 1 у Ozon нет пользовательски изменяемых параметров расчёта скидок. Параметры WB не переносятся на Ozon.
