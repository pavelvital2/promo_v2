# WB_DISCOUNTS_EXCEL_SPEC.md

Трассировка: ТЗ §14-§15, §17, §24.

## Назначение

WB -> Скидки -> Excel рассчитывает итоговую скидку и записывает её в колонку `Новая скидка`.

## Входной состав

Один WB run принимает:

- ровно 1 файл цен;
- от 1 до 20 файлов акций;
- только `.xlsx`;
- максимум 25 МБ на один файл;
- максимум 100 МБ суммарно на run;
- если лист один - используется он;
- если листов несколько - используется первый лист.

## Обязательные колонки

Файл цен:

- `Артикул WB`;
- `Текущая цена`;
- `Новая скидка`.

Файлы акций:

- `Артикул WB`;
- `Плановая цена для акции`;
- `Загружаемая скидка для участия в акции`.

## Нормализация

Для `Артикул WB`:

- привести к строке;
- trim;
- удалить суффикс `.0`;
- пустое значение считать отсутствующим.

Для числовых значений:

- удалить обычные пробелы;
- удалить неразрывные пробелы;
- заменить запятую на точку;
- нечисловое значение считать отсутствующим.

Одинаковая нормализация обязательна в check и process.

## Агрегация акций

Единица агрегации: нормализованный `Артикул WB`.

По каждому артикулу:

- `min_discount = MIN(Загружаемая скидка)`;
- `max_plan_price = MAX(Плановая цена для акции)`.

В агрегации участвуют только валидные строки валидных promo files.

## Формула

```text
calculated_discount = ceil((1 - max_plan_price / current_price) * 100)
```

Требования:

- запрещено использовать float;
- используется точная десятичная арифметика;
- округление только `ceil`.

## Гибридная логика

Сначала:

```text
final_discount_pre_threshold = MIN(min_discount, calculated_discount)
```

Далее строго в указанном порядке:

1. Если артикул отсутствует во всех валидных файлах акций: `final_discount = fallback_no_promo_percent`.
2. Иначе если `final_discount_pre_threshold > threshold_percent`: `final_discount = fallback_over_threshold_percent`.
3. Иначе: `final_discount = final_discount_pre_threshold`.

Порядок менять нельзя.

## Ограничения результата

Итоговая скидка:

- всегда целое число;
- диапазон 0-100 включительно;
- выход за диапазон 0..100 является error строки.

Если рассчитанный `final_discount` меньше 0 или больше 100:

- строка получает error;
- reason/result code = `wb_discount_out_of_range`;
- check завершается с ошибками;
- process запрещён;
- значение не обрезается до 0..100;
- частичная обработка workbook запрещена.

## Управляемые параметры WB

Только эти параметры управляемы на этапе 1:

- `wb_threshold_percent`;
- `wb_fallback_no_promo_percent`;
- `wb_fallback_over_threshold_percent`.

Параметры относятся к магазину/кабинету. Каскад: store value -> system default. Операция хранит applied value и source.

Системные defaults этапа 1:

| Parameter code | System default |
| --- | --- |
| `wb_threshold_percent` | `70` |
| `wb_fallback_over_threshold_percent` | `55` |
| `wb_fallback_no_promo_percent` | `55` |

Запрещено добавлять иные управляемые WB-параметры без утверждения.

## Check WB

Check обязан:

- проверить состав файлов;
- открыть и провалидировать файл цен и promo files;
- выполнить нормализацию;
- выявить невалидные строки;
- агрегировать валидные акции;
- полностью промоделировать результат по каждой строке;
- сформировать summary audit;
- сформировать detail audit;
- сформировать краткий итог;
- не создавать output file;
- ничего не записывать в workbook.

## Process WB

Process обязан:

- выполняться только по допустимой check-основе;
- не выполняться, если check-основа содержит errors;
- учитывать актуальность проверки;
- выполнить полный расчёт;
- изменить только колонку `Новая скидка`;
- сохранить output workbook;
- сформировать summary audit;
- сформировать detail audit;
- сформировать краткий итог.

## Запреты workbook

Нельзя:

- менять любые колонки, кроме `Новая скидка`;
- менять порядок строк;
- разрушать формулы;
- разрушать пригодность workbook к дальнейшему использованию;
- использовать float в финальном расчёте.

## Критические ошибки

Минимально:

- не загружен файл цен;
- не загружен ни один файл акций;
- файл не открывается как `.xlsx`;
- книга повреждена;
- нет обязательной колонки файла цен;
- все файлы акций невалидны;
- дубли в файле цен по `Артикул WB`;
- невозможность безопасной записи в `Новая скидка`.

## Detail audit

По каждой строке фиксируется:

- row number;
- normalized `Артикул WB`;
- current price;
- aggregate promo values, если применимо;
- final_discount_pre_threshold, если применимо;
- final_discount;
- reason/result code;
- errors/warnings;
- human-readable explanation.

## WB reason/result code catalog

Закрытый перечень codes этапа 1:

| Code | Когда применяется |
| --- | --- |
| `wb_valid_calculated` | строка price file валидна, promo data найдены, итоговая скидка рассчитана без замены fallback из-за отсутствия акции или превышения порога |
| `wb_no_promo_item` | артикул отсутствует во всех валидных promo files, применён `wb_fallback_no_promo_percent` |
| `wb_over_threshold` | `final_discount_pre_threshold > wb_threshold_percent`, применён `wb_fallback_over_threshold_percent` |
| `wb_missing_article` | в строке отсутствует валидный `Артикул WB` |
| `wb_invalid_current_price` | в строке отсутствует или невалидна `Текущая цена`, либо цена не допускает безопасный расчёт |
| `wb_duplicate_price_article` | в price file найден дубль нормализованного `Артикул WB` |
| `wb_missing_required_column` | в price file или promo file отсутствует обязательная колонка |
| `wb_invalid_promo_row` | promo row не участвует в агрегации из-за невалидного артикула или числового значения |
| `wb_invalid_workbook` | workbook не открывается, повреждён, имеет недопустимый формат или не может быть безопасно прочитан |
| `wb_output_write_error` | process не может безопасно записать output workbook в колонку `Новая скидка` |
| `wb_discount_out_of_range` | итоговая скидка вне диапазона 0..100; row error, check с ошибками, process запрещён |

Добавление или переименование WB reason/result codes требует утверждённого изменения документации/ADR.
