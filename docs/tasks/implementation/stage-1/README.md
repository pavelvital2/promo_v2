# README.md

Трассировка: ТЗ §25-§26.

## Как читать задачи

Каждый файл `TASK-*.md` является самостоятельной task-scoped постановкой для будущего агента реализации. Агент читает:

- конкретный файл задачи;
- `AGENTS.md`;
- `docs/tasks/implementation/stage-1/IMPLEMENTATION_TASKS.md`;
- документы, ADR/GAP и разделы ТЗ, перечисленные в задаче;
- связанные записи `docs/gaps/GAP_REGISTER.md`, если они указаны как blockers или constraints.

Полное перечитывание итогового ТЗ на каждую задачу не требуется. Если перечисленного контекста недостаточно, агент не домысливает правило, а фиксирует gap/blocker и передаёт вопрос оркестратору.

## Формат handoff

Каждая задача должна завершаться отчётом:

- что сделано;
- изменённые файлы;
- закрытые требования ТЗ;
- использованные входные документы / разделы ТЗ;
- выполненные проверки;
- найденные проблемы;
- новые или закрытые gaps;
- вопросы для эскалации заказчику через оркестратора;
- следующий рекомендуемый шаг.

## Общие запреты

- Не менять итоговое ТЗ.
- Не создавать прикладной код вне разрешённых областей задачи.
- Не смешивать операции, аудит и технический журнал.
- Не менять утверждённый стек ADR-0006 без нового ADR.
- Не изменять seed-модель прав ADR-0007 без решения заказчика.
- Не изменять формат visible identifiers ADR-0008 без решения заказчика.
- Не закрывать downstream gaps предположениями.
